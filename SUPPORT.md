# Support

Use the right support channel so questions and issues are easier to handle.

## Where to Go

### GitHub Discussions

Use Discussions for:

- product usage questions
- setup help
- architecture discussion
- feature ideas before implementation
- community sharing

Discussion entry:

- `https://github.com/codeking-ai/cligate/discussions`

### GitHub Issues

Use Issues for:

- confirmed bugs
- broken setup flows
- release or packaging problems
- clear documentation defects
- actionable feature requests

Issue entry:

- `https://github.com/codeking-ai/cligate/issues`

### Security Reports

For token leaks, credential exposure, or other sensitive problems, do not open a public issue first. Follow [SECURITY.md](./SECURITY.md).

## Before Opening an Issue

Please include:

- CliGate version
- installation method
- operating system
- affected area
- steps to reproduce
- relevant logs with secrets removed

## Documentation First

Before asking a product-usage question, check:

- [README.md](./README.md)
- [docs/README.md](./docs/README.md)
- [docs/product-manual.en.md](./docs/product-manual.en.md)
- [docs/product-manual.zh-CN.md](./docs/product-manual.zh-CN.md)

## Release and Version Questions

If the version shown on GitHub Releases, npm, and the local repository do not match, include:

- the Git tag you expected
- the package version from `package.json`
- the command you ran
- the exact error message or missing artifact
